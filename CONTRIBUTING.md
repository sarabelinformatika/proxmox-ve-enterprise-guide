# Contributing

Thank you for your interest in contributing to the Proxmox VE Enterprise Guide.

This project aims to become a comprehensive enterprise knowledge base covering Proxmox VE architecture, operations, security, and best practices.

---

## How You Can Contribute

Contributions are welcome in the following areas:

- Documentation improvements
- Grammar corrections
- Technical corrections
- Clarifying existing content
- Updating outdated information
- Suggesting additional enterprise best practices

---

## Before Submitting

Please ensure that your contribution:

- follows the writing style of the existing documentation;
- focuses on enterprise environments;
- remains vendor-neutral where possible;
- avoids unnecessary complexity;
- uses clear and professional technical language.

---

## Pull Requests

Please include:

- a clear description of the proposed change;
- the reason for the change;
- supporting references when applicable.

Small, focused pull requests are preferred over large changes.

---

## Issues

Bug reports and documentation improvements are always appreciated.

When opening an issue, include:

- affected chapter;
- description;
- proposed improvement (if applicable).

---

Thank you for helping improve this documentation.
